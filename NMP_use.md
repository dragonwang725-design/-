# 空模型范式（NMP）在智能机器人与智能自动驾驶领域的应用

## 一、核心问题：为什么当前具身智能不可靠

当前智能机器人和自动驾驶领域，正陷入一场"能力幻觉"：

- **机器人**：能跳舞、能翻跟头，但进不了厨房帮忙——因为"没见过"你的厨房布局
- **自动驾驶**：高速上流畅如老司机，但遇到施工路段就懵——因为"训练数据没覆盖"


**不是"能力不够"，是"结构不对"。**

当前方案（端到端神经网络、世界模型）的共同结构：
- 传感器全量输入 → 模型推理 → 直接输出动作
- 模型是主体，没有外部约束

**结果：** 训练数据覆盖的场景表现好，没见过的场景"猜"——猜对了是流畅，猜错了是事故。



| 当前做法         | 问题          | NMP 的视角                          |
| ------------ | ----------- | -------------------------------- |
| 把视频帧全部输入视觉模型 | 信息过载，噪声淹没信号 | LLM 是"抛开事实讲逻辑"，视觉模型也一样——它不知道什么重要 |
| 提取所有物体、所有关系  | 计算浪费，延迟爆炸   | 没有"目的"的过滤，全量提取 = 无效提取            |
| 构建完整场景图      | 存储爆炸，检索困难   | "知识"不是"事实"——场景图是知识，不是与目的相关的事实    |

全视频解读 = 让模型"自己猜什么重要" = 幻觉的温床。

**NMP 的改动：** 在"模型推理"和"输出动作"之间，加一层**事实校验**。

| 步骤             | NMP 对应      | 作用               |
| -------------- | ----------- | ---------------- |
| 视频 → 文本（结构化描述） | 探针采集        | 把连续信号变成离散事实      |
| 目的决定提取什么       | L2 空模型意图识别  | 不是"全量理解"，是"按需提取" |
| 只抓取与目的相关的事实    | 元事实库检索      | 事实主权：用户/系统决定什么重要 |
| 基于提取的事实生成策略    | L3 LLM 约束生成 | 模型只组织语言，不决定边界    |

**示例**：

场景：机器人进入厨房，目的"拿杯子倒水"

> 全视频解读：
- 识别：冰箱、微波炉、灶台、水槽、橱柜、杯子、水壶、窗户、灯光、地板...
- 关系：杯子在橱柜上，水壶在水槽边，微波炉在灶台上方...
- 问题：信息太多，模型需要"推理"什么重要 → 可能推理错

> NMP 方式：
- 目的："拿杯子倒水"
- L2 提取相关事实：
  - 杯子位置（元事实库：厨房物品坐标）>
  - 水壶状态（元事实库：是否有水、温度）
  - 可达性（元事实库：路径是否畅通）
- L3 生成策略：基于以上事实，规划动作

不提取的：
- 冰箱品牌（无关）
- 窗户颜色（无关）
- 灯光亮度（除非影响识别）

## 二、NMP 在具身智能中的具体结构

### 2.1 三层分工（接口定义）

| 层级 | 输入 | 输出 | 具体实现 |
|------|------|------|---------|
| **L1** | 人类语音/系统信号 | 结构化意图 | 意图识别模型（轻量）或规则解析 |
| **L2** | 意图 + 传感器数据 | 事实包 + 约束集 | 规则引擎 + 检索系统（见下文） |
| **L3** | 事实包 + 约束集 | 策略文本/动作序列 | 调用现有 LLM/视觉模型 API |

## 二、NMP 在具身智能中的具体结构

### 2.1 三层分工的具身化

| 层级 | 名称 | 具身智能中的角色 | 权限 |
|------|------|----------------|------|
| **L1** | 人类/系统意图 | 设定任务目标："拿杯子倒水""高速巡航保持车道" | 不可委托 |
| **L2** | 空模型（NMP） | **事实提取器 + 约束锁定器** | 唯一物理执行权 |
| **L3** | 全模型（LLM/视觉模型） | **策略生成器** | 零写入、零执行、零计划修改 |

**关键接口：**
```
L1 → L2: {purpose: "倒水", target: "用户", location: "客厅"}
L2 → L3: {facts: [...], constraints: [...], cases: [...]}
L3 → L2: {strategy: "走到厨房→取杯→接水→递送"}
L2 → 执行器: 校验通过 → 下发动作；校验失败 → 拦截/降级

```
**关键差异：**
- 当前方案：摄像头/激光雷达 → 端到端模型 → **直接输出动作**（模型即主体）
- NMP 方案：传感器 → **L2 事实提取** → **目的过滤** → **约束锁定** → L3 策略生成 → **L2 验证执行**（外壳即主体）

### 2.2 两大核心库的具身化

#### 元事实库：机器人的"身体记忆"

不是"知识图谱"，是"**确定性事实的本地寄存器**"。

| 场景 | 元事实库内容 | 来源 |
|------|------------|------|
| 家庭机器人 | 家具坐标、物品位置、用户偏好、安全边界 | 用户确认、传感器标定 |
| 工业机器人 | 设备参数、操作规范、环境阈值、工位布局 | 企业确认、系统配置 |
| 自动驾驶 | 高精地图、车道类型、限速值、法规条款 | 权威发布、实时校验 |

**关键属性：**
- **本地**：存于设备端，非云端
- **确定性**：用户/企业确认，非模型推断
- **时态化**：带时间戳，可更新
- **目的驱动**：按当前任务动态加载子集

#### 判例库：具身智能的"疼痛记忆"

不是"错误日志"，是"**否定性知识的免疫档案**"。

| 场景 | 判例库内容 | 价值 |
|------|----------|------|
| 机器人 | "雨天瓷砖地打滑跌倒""黑暗环境误判物体距离" | 同类场景预判拦截 |
| 自动驾驶 | "雨天弯道侧翻""逆光行人误判""施工路段跟车过近追尾" | 全域车辆实时免疫 |

**关键机制：**
- **结构化**：不是文本描述，是可匹配的约束模式
- **可泛化**：从具体事故提取抽象模式（"湿滑地面+急转→降速"）
- **实时同步**：单点事故 → 结构化入库 → 同型号全域免疫

## 三、机器人领域的应用：从"炫技"到"可用"

### 3.1 当前痛点：为什么机器人进不了家门

| 问题 | 根因 | NMP 的解 |
|------|------|---------|
| "没见过你家厨房" | 训练数据不包含用户私域环境 | 元事实库：用户确认的家具/物品坐标 |
| "碰倒了花瓶" | 模型"推理"出空间关系，但推理错了 | 判例库："狭窄空间+转身→先确认后方" |
| "用户说'拿那个'，机器人懵了" | 语言指称无锚定 | 元事实库："那个"= 用户视线方向+最近物体 |
| "OTA 更新后行为变了" | 模型权重黑箱更新 | 元事实库不变，换模型不影响行为边界 |

### 3.2 NMP 机器人架构

```
用户指令："帮我倒杯水"
    ↓
L2 空模型：
  - 目的解析："倒水" = 取杯子 + 接水 + 递送
  - 事实提取（元事实库）：
    - 杯子位置：厨房橱柜第二层
    - 水壶状态：客厅茶几，有水，温度 45°C
    - 用户位置：沙发
    - 路径状态：地面无障碍
  - 约束锁定：
    - 不可触碰：热水壶（高温）
    - 不可进入：卧室（隐私区域）
    - 必须轻拿：玻璃杯（易碎）
    ↓
L3 视觉-语言模型：
  - 基于以上事实，生成动作序列
  - "走到厨房 → 打开橱柜 → 取杯子 → 走到客厅 → 拿起水壶 → 倒水 → 走到沙发 → 递送"
    ↓
L2 验证：
  - 每步动作前，校验事实一致性
  - 发现"水壶温度 85°C"（事实更新）→ 触发判例库"高温液体→不可直接递送"
  - 拦截原策略，生成新策略："等待降温至 50°C 以下再递送"
    ↓
执行
```

### 3.3 关键价值：用户主权

| 当前方案 | NMP 方案 |
|---------|---------|
| 机器人厂商拥有"智能" | 用户拥有"事实"——机器人只是临时工 |
| 换机器人 = 重新训练适应 | 换机器人 = 导入元事实库，行为一致 |
| 隐私数据上传云端 | 元事实库本地，隐私不外泄 |

## 四、自动驾驶领域的应用：从"流畅"到"可信"

### 4.1 当前痛点：为什么 L3/L4 不敢落地

| 问题 | 根因 | NMP 的解 |
|------|------|---------|
| 测了 10 亿公里还是出事 | 长尾场景无限，测试覆盖不了 | 判例库："错一次，全域免疫" |
| 事故后黑箱说不清 | 端到端模型不可解释 | 每一步可追溯：事实→约束→策略→执行 |
| OTA 更新怕"越修越坏" | 模型权重黑箱变化 | 元事实库不变，约束层不变，换模型不影响安全边界 |
| 法规要"可解释" | 模型给不出 | 裁决日志：基于哪条事实、哪条判例、哪个阈值 |

### 4.2 NMP 自动驾驶架构

```
场景：高速巡航，目的"保持车道，安全跟车"
    ↓
L2 空模型：
  - 事实提取（元事实库）：
    - 当前车道：左侧车道，限速 120km/h
    - 前车状态：距离 80m，速度 110km/h
    - 天气：小雨，路面湿滑系数 0.7
    - 车辆状态：轮胎磨损度 15%，制动正常
  - 判例库匹配：
    - "雨天+高速+跟车距离<100m+路面湿滑" → 触发判例 "侧翻风险"
    - 历史阈值：同类场景建议限速 80%，跟车距离 ≥120m
  - 约束锁定：
    - 目标速度：≤96km/h（限速 80%）
    - 跟车距离：≥120m
    - 禁止：急变道、急加速
    ↓
L3 端到端模型：
  - 基于以上约束，生成平滑驾驶策略
  - 油门/刹车/转向的连续控制信号
    ↓
L2 验证：
  - 实时校验：策略输出是否违反约束
  - 发现"模型建议变道超车"（违反"禁止急变道"约束）
  - 拦截 → 降级为保守策略 → 告警人工接管
    ↓
执行 / 人工接管
```

### 4.3 关键价值：可审计的安全

| 当前方案 | NMP 方案 |
|---------|---------|
| 事故后查日志，黑箱 | 事故后查裁决日志：基于哪条事实、哪条判例、哪个阈值 |
| 责任归属模糊 | 责任清晰：事实库谁维护、判例库谁更新、约束谁锁定 |
| 保险不敢保 | 可审计 → 可定价 → 可保险 |

## 五、与现有方案的对比

| 维度 | 端到端（特斯拉 FSD） | 世界模型（Wayve/GAIA-1） | NMP |
|------|---------------------|------------------------|-----|
| **智能来源** | 神经网络内化驾驶直觉 | 生成式仿真预测未来 | 本地事实 + 判例约束 |
| **事实锚定** | 训练数据（历史视频） | 仿真生成（虚拟场景） | 元事实库（实时、本地、确定性） |
| **错误处理** | 长尾场景靠"见多识广" | 靠仿真覆盖 | 判例库"错一次，全域免疫" |
| **可解释性** | 黑箱 | 黑箱 | 白箱：事实→约束→策略→执行 |
| **主权归属** | 车企拥有模型 | 车企拥有仿真 | 用户/企业拥有事实库 |
| **与 LLM 关系** | LLM 是主体 | LLM 是主体 | LLM 是策略生成器，外壳是主体 |

**NMP 不是替代现有方案，是给现有方案戴"紧箍咒"：**
- 端到端模型负责"流畅性"
- NMP 约束层负责"确定性"
- 两者叠加 = "既流畅又可信"的自动驾驶

## 六、工程落地路径

### 6.1 当前可做的（V1.0）

| 模块 | 技术 | 工作量 |
|------|------|--------|
| 元事实库 | 本地 SQLite/JSON + 向量检索 | 1 人周 |
| 判例库 | 结构化错误记录 + 精确匹配 | 1 人周 |
| L2 空模型 | Python 规则引擎 + 字符串匹配 | 1 人周 |
| L3 接口 | 调用现有 LLM/视觉模型 API | 1 人天 |
| 验证层 | 简单对比实验 | 1 人周 |

**合计：1 个月，1 个工程师，可出概念验证。**

### 6.2 下一步（V2.0）

| 模块 | 技术挑战 |
|------|---------|
| 判例库泛化匹配 | 向量相似度 + 模式抽象 |
| 动态阈值裁决 | 贝叶斯推断 / 小模型辅助 |
| 实时事实采集 | 传感器融合 + 结构化提取 |
| 多车判例同步 | 边缘节点聚合 + 脱敏同步 |

### 6.3 终极形态（V3.0）

- **元事实库**：全车/全场景的事实主权寄存器
- **判例库**：行业级错误免疫网络
- **裁决 AI**：情境化动态阈值，可审计可追溯
- **L2 空模型**：硬件级实时约束引擎（<10ms）

## 七、核心公理：具身智能不需要"意识"，需要"敬畏"

> **自主意识 = 目的 + 判断**

| 成分 | 所有者 | 约束 |
|------|--------|------|
| **目的** | **人类** | 不可更改。不可计算。 |
| **判断** | **机器** | 可计算的。不可逆的。但**必须基于事实**。 |

**具身智能的可靠性，不在于机器"多聪明"，在于机器"多不敢乱来"。**

空模型范式（NMP）不是让机器人/自动驾驶"更像人"，是让它们"**更知道自己是机器**"——
- 没有身体，所以必须依赖本地事实库
- 没有疼痛记忆，所以必须依赖判例库
- 没有常识，所以必须在约束边界内行动

**这不是限制，是解放。**

解放了人类对"机器会不会失控"的恐惧，
解放了机器对"我必须全知全能"的幻觉。
